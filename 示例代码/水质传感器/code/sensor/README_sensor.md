# ws63 I2C 传感器数据显示例程 (OLED)

## 1. 概述

本项目代码 (`sensor.c`) 是一个针对 **ws63** 系列单片机的综合传感器演示例程。其核心功能是通过 **I2C** 总线与多个传感器和一个 **SSD1306 OLED 显示屏** 进行通信。

代码会初始化 I2C 接口，然后周期性地从 **pH、温度、TDS 和浊度传感器** 读取数据，并将这些实时数据格式化后显示在 OLED 屏幕上，为用户提供一个直观的本地数据显示方案。

## 2. 主要功能

- **I2C 主机模式**: 将 ws63 单片机配置为 I2C 总线的主机。
- **多传感器驱动**:
    - 初始化并读取 pH 传感器数据。
    - 初始化并读取温度传感器数据。
    - 初始化并读取 TDS (总溶解固体) 传感器数据。
    - 初始化并读取浊度 (TS) 传感器数据。
- **OLED 显示**:
    - 初始化 SSD1306 OLED 显示屏。
    - 在屏幕上清晰地展示四种传感器的实时数值。
- **参数可配置**: 用户可以通过修改宏定义来配置：
    - I2C 总线 ID (例如 I2C1)
    - I2C SCL (时钟) 和 SDA (数据) 引脚
    - I2C 通信速率
- **任务化运行**: 创建一个独立的后台任务来执行传感器数据读取和屏幕刷新，不影响其他系统功能。

## 3. 硬件要求

- **主控芯片**: ws63 系列单片机开发板。
- **显示屏**: **SSD1306** 驱动的 I2C OLED 显示屏。
- **传感器 (均为 I2C 接口)**:
    - pH 传感器模块
    - 温度传感器模块
    - TDS 传感器模块
    - 浊度 (TS) 传感器模块
- **连接**: 所有传感器和 OLED 屏幕都需并联到同一组 I2C 总线上 (SCL 接 SCL, SDA 接 SDA)。

## 4. 如何配置

**在使用前，请根据您的硬件连接修改 I2C 引脚配置！**

1.  打开 `sensor.c` 文件。
2.  定位到文件头部的配置区域：

    ```c
    #define CONFIG_I2C_MASTER_BUS_ID          1
    #define CONFIG_I2C_SCL_MASTER_PIN         15
    #define CONFIG_I2C_SDA_MASTER_PIN         16
    ```

3.  **修改引脚**:
    - `CONFIG_I2C_MASTER_BUS_ID`: I2C 总线号，根据您的硬件设计选择，例如 `1` 代表 I2C1。
    - `CONFIG_I2C_SCL_MASTER_PIN`: 修改为您的单片机 **SCL** 引脚所连接的 **GPIO 编号**。
    - `CONFIG_I2C_SDA_MASTER_PIN`: 修改为您的单片机 **SDA** 引脚所连接的 **GPIO 编号**。

4.  **（可选）修改其他参数**:
    - `I2C_SET_BAUDRATE`: I2C 通信速率，默认为 `400000` (400kHz)。
    - `I2C_TASK_DURATION_MS`: 屏幕刷新和数据采集的周期，默认为 `1000` (1秒)。

## 5. 如何编译和使用

1.  **完成配置**: 确保已根据第 4 节的说明正确配置了 I2C 引脚。
2.  **确认依赖**: 确保您的工程中已经包含了以下驱动库：
    - `ssd1306.c` / `ssd1306.h` (OLED 驱动)
    - `ph_demo.c`, `tem_demo.c`, `conductance_demo.c`, `turbidity_demo.c` (各传感器驱动)
3.  **集成到工程**:
    - 将 `sensor.c` 文件放置在您项目的 `src/application/samples/peripheral/sensor/` 目录下。
    - 确保您项目的 `BUILD.gn` 或 `CMakeLists.txt` 文件包含了此源文件以及所有依赖的驱动文件，以便它们能被编译进最终的固件中。
4.  **编译和烧录**: 使用您项目的标准流程编译整个工程，并将生成的固件烧录到 ws63 开发板中。
5.  **连接硬件**: 将所有 I2C 设备的 SCL 引脚连接到 ws63 配置的 SCL 引脚，SDA 引脚连接到 SDA 引脚，并确保所有设备共地和正常供电。
6.  **运行和查看**: 开发板上电后，代码会自动运行。OLED 屏幕上应该会开始周期性地显示 "PH:", "TDS:", "NTU:", "TEMP:" 以及对应的实时数据。

## 6. 注意事项

- **I2C 地址**: 本例程中 `I2C_MASTER_ADDR` 设置为 `0x0`，通常在主机模式下无需指定从机地址，而是在调用 `uapi_i2c_writeread` 等函数时传入。请确保您的传感器驱动和 OLED 驱动中使用了正确的设备 I2C 地址。
- **上拉电阻**: I2C 总线通常需要在 SCL 和 SDA 线上各接一个上拉电阻（例如 4.7kΩ）到 VCC。本代码中通过 `uapi_pin_set_pull` 启用了内部上拉，但在某些情况下，外部上拉电阻会使通信更稳定。
- **传感器驱动**: 本代码依赖 `PH_GetData()` 等函数来获取数据。您需要确保这些函数在对应的驱动文件中已正确实现。
