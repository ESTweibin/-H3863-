# ws63 水质监测数据通过串口上报华为云 IoT 平台

## 1. 概述

本项目代码 (`mqtt_huawei_cloud_sender.c`) 专为 **ws63** 系列单片机设计。其核心功能是 **采集多种水质传感器的数据**（如 pH 值、温度、TDS、浊度），然后通过 **UART 串口** 将这些数据发送给一个外接的 **4G 通信模块**。

4G 模块接收到数据后，会将其以 **JSON 格式** 上报到 **华为云物联网（IoTDA）平台**，从而实现远程设备监控和数据分析。

此代码实现了数据采集与通信上报的解耦，并提供了易于修改的配置项，方便用户根据自己的硬件进行适配。

## 2. 主要功能

- **定时采集**: 周期性地从传感器获取 pH、温度、TDS 和浊度数据。
- **JSON 格式化**: 将采集到的数据封装成符合华为云物模型要求的标准 JSON 字符串。
- **串口发送**: 通过指定的 UART 端口，将 JSON 数据字符串发送出去。
- **参数可配置**: 用户可以轻松修改宏定义来配置：
    - UART 端口号 (UART0 / UART1)
    - UART TX/RX 引脚
    - 引脚功能模式
    - 数据上报周期
    - 串口波特率
- **任务化运行**: 创建一个独立的后台任务来执行数据采集和发送，不阻塞主流程。

## 3. 硬件要求

- **主控芯片**: ws63 系列单片机开发板。
- **传感器**:
    - pH 传感器
    - 温度传感器
    - TDS (总溶解固体) 传感器
    - 浊度 (TS) 传感器
    - *（注意：代码中 `PH_GetData()` 等函数为示例，您需要将其替换为实际驱动您传感器的代码）*
- **通信模块**: 支持 **串口（TTL电平）** 通信的 **4G DTU** 或 **4G Cat.1/Cat.M 模块**。该模块需要预先配置好华为云的接入信息。

## 4. 如何配置

**在使用前，您必须根据您的硬件连接修改引脚配置！**

1.  打开 `mqtt_huawei_cloud_sender.c` 文件。
2.  定位到文件头部的配置区域：

    ```c
    // =================================================================================
    // =================== 请根据您的硬件修改以下引脚配置 ===================
    // =================================================================================
    #define CONFIG_UART_BUS_ID 0 // 指定使用UART0，如需UART1请改为1
    #define CONFIG_UART_TXD_PIN 9 // <<<--- 修改这里: 设置正确的TXD引脚编号
    #define CONFIG_UART_TXD_PIN_MODE PIN_MODE_1 // <<<--- 修改这里: 设置正确的引脚模式 (例如 PIN_MODE_1)
    #define CONFIG_UART_RXD_PIN 10 // <<<--- 修改这里: 设置正确的RXD引脚编号
    #define CONFIG_UART_RXD_PIN_MODE PIN_MODE_1 // <<<--- 修改这里: 设置正确的引脚模式 (例如 PIN_MODE_1)
    // =================================================================================
    ```

3.  **修改引脚**:
    - `CONFIG_UART_TXD_PIN`: 修改为您的单片机 **TXD** 引脚所连接的 **GPIO 编号**。
    - `CONFIG_UART_RXD_PIN`: 修改为您的单片机 **RXD** 引脚所连接的 **GPIO 编号**。
    - `CONFIG_UART_TXD_PIN_MODE` / `CONFIG_UART_RXD_PIN_MODE`: 根据芯片数据手册，设置引脚对应的 **UART 功能模式**。

4.  **（可选）修改其他参数**:
    - `UART_SEND_INTERVAL_MS`: 数据上报的时间间隔（单位：毫秒），默认为 `5000` (5秒)。
    - `UART_BAUDRATE`: 串口波特率，默认为 `115200`。**此参数必须与您的 4G 模块的串口波特率设置完全一致**。

## 5. 如何编译和使用

1.  **完成配置**: 确保已根据第 4 节的说明正确配置了 UART 引脚。
2.  **替换传感器驱动**: 将代码中 `PH_GetData()`, `TEMP_GetData()`, `TDS_GetData_PPM()`, `TS_GetData()` 这几个 **占位函数** 替换为您自己编写的、能够从真实传感器读取数据的函数。
3.  **集成到工程**:
    - 将 `mqtt_huawei_cloud_sender.c` 文件放置在您项目的 `src/application/samples/peripheral/mqttconnect/` 目录下。
    - 确保您项目的 `BUILD.gn` 或 `CMakeLists.txt` 文件包含了此源文件，以便它能被编译进最终的固件中。
4.  **编译和烧录**: 使用您项目的标准流程编译整个工程，并将生成的固件烧录到 ws63 开发板中。
5.  **连接硬件**: 将 4G 模块的 `RXD` 连接到 ws63 的 `TXD` 引脚，4G 模块的 `TXD` 连接到 ws63 的 `RXD` 引脚，并共地。
6.  **运行和调试**:
    - 开发板上电后，代码会自动运行。
    - 您可以在华为云物联网平台的设备页面，查看设备是否在线以及 "设备影子" 或 "实时数据" 中是否有新的数据上报。
    - 如果没有数据，请检查：
        - 串口引脚配置和物理连接是否正确。
        - 单片机与 4G 模块的波特率是否一致。
        - 4G 模块本身是否工作正常并已成功连接到网络和华为云。

## 6. JSON 数据格式说明

代码生成的 JSON 格式如下，严格遵循华为云物模型定义：

```json
{
    "services": [
        {
            "service_id": "ws63",
            "properties": {
                "temp": 25.50,
                "ph": 7.00,
                "TDS": 120.00,
                "TS": 30.00
            }
        }
    ]
}
```

- `service_id`: 必须与您在华为云上定义的 **服务ID** 一致。
- `properties` 内的 `temp`, `ph`, `TDS`, `TS` 字段名也必须与您定义的 **属性ID** 完全一致。

如果平台报告 JSON 格式错误，请仔细核对以上字段。
